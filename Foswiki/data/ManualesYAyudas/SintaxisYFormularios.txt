---+ Que es y para que se usan

   * Son formularios HTML que se definen en un TOPIC. 
      * text
      * select
      * radiobuttons
      * checkbox
      * textarea
      * date
   * Se utilizan para estructura el contenido dentro Foswiki

---+ Como se define un formulario

   * Se debe crear un platilla que especifique el formulario
   * La plantilla es un topic que contiene un tabla.
   * Las filas de la tabla definen los campos del formulario
   * (Sandbox.FormularioDePrueba)

---+ Como se habilitan los formularios

   * Se utiliza la variable WEBFORMS en WebPreferences para definir los formularios.
   * Cuando un topic no tiene formulario en la acción EDIT se puede asociar un formulario habilitado.
   * Si el topic ya tiene un formulario se le puede cambiar.

---+ Como se le asigna un formulario a un topic

   * Editando el topic y utilizar la utilidad de "add form" o "change form"
   * Creando un topic que defina un formulario HTML que permita crear nuevos topics utilizando una plantilla y un formulario especifico
   * (Sandbox.CrearRegistroPrueba)

---+ Cambios en los formularios

   * Los cambios sobre un formulario serian: 
      * Agregar campos
      * Eliminar campos
      * Cambiar el nombre a un capo
   * (Sandbox.FormularioDePrueba)

---+ Estructura de una plantilla de formulario

   * Es un topic con una tabla con una estructura definida
   * La tabla debe contener los siguientes campos en el mismo orden:
<literal>| Name | Type | Size | Value | Tooltip message | Attributes |</literal>
   * Los tipos de dato son:
<form action=""> 
   * Text: <input type="text" name="Text" size="40" />
   * Text area: <textarea name="TextArea" col=5 row=40></textarea>
   * Select: <select name="Select" size="1"><option value="-">-</option><option value="1">1</option></select>
   * Radio Buttons: <input type="radio" name="Radio" value="yes">Yes <input type="radio" name="Radio" value="No">No
   * CheckBox: <input type="checkbox" name="Checkbox">RCCS</input>
   * Date
   * Label
</form>

---+ Valores de campos en otros topics

   * Se puede definir un topic con los valores para los campos tipo select, checkbox, radio buttons.
   * Se debe crear el topic con la siguientes estructura:
<literal>| Name | Type | Tooltip message |</literal>
   * En este topic se pueden realizar búsquedas con SEARCH

---+ Resumiendo - Un registro debe tener

   * Una definición
   * Una Web que los agrupe
   * Una plantilla
   * Un topic que permita crearlos
   * Un topic que los consolide
